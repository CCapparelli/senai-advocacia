Próximos passos

Como por exemplo:

Componeitização dos formulários

Os formulários (login, signup e contato) podem ser "quebrados" em unidades (componentes) menores, a fim de criar (inclusive) uma "identidade visual" para o site - ao invés de encontrar diferentes estilos e comportamentos.

Componeitização das "páginas" internas / protegidas [como esta]

Separar os menus da área principal, para otimizar o reuso, a manutenção e a implantação de novos componentes e funcionalidades - ao invés de ficar copiando-e-colando (e atualizando componentes pré-desenvolvidos) toda vez que um novo recurso é introduzido.
